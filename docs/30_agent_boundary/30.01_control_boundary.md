# Human-Agent-Discord Control Boundary

This repository creates a control boundary between human intent, Agent
projection, HCL-managed Discord state, and live Discord behavior.

## Boundary Model

```text
Human request
  -> Wendao BPMN workflow / petgraph topology
  -> knowledge_operation or intent manifest
  -> Agent projection
  -> OpenTofu plan or runtime action
  -> Discord state or message
  -> receipt
```

The Agent is a semantic compiler, not an unrestricted Discord operator. It
translates user intent and Wendao knowledge context into reviewable changes.

## Boundary Objects

| Object | Purpose |
|---|---|
| Operation | Discord projection envelope for one Wendao workflow step. |
| Intent | What the human wants to happen. |
| Policy | What can be planned, auto-applied, or must be reviewed. |
| Projection | The translation from intent to HCL variables or runtime action. |
| Receipt | The record of what happened and which knowledge/source context caused it. |
| Workflow ref | Opaque BPMN/petgraph reference copied from Wendao into receipts. |

## Wendao Context

Wendao should provide knowledge context such as paper metadata, summary, tags,
source references, retrieval evidence, BPMN node refs, and petgraph node refs.
This repository should not depend on Wendao internals. The bridge should be an
operation, manifest, or receipt contract that can be produced by Wendao, an
Agent, or a user.

## Discord Boundary

Discord remains the live human discussion venue. The HCL library governs stable
structure and durable objects. Runtime tools or bots may handle threads, posts,
and replies when those are too dynamic for OpenTofu state.
