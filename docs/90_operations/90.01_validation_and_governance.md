# Validation and Governance

This project uses a small command surface around OpenTofu and the Johnny.Decimal
documentation topology.

## OpenTofu Validation

```shell
direnv exec . just check
```

Expanded commands:

```shell
direnv exec . just fmt
direnv exec . just init-example
direnv exec . just validate-example
git diff --check
```

## Live Discord Check

Live plans and applies use `TF_VAR_DISCORD_BOT_TOKEN` plus
`TF_VAR_DISCORD_SERVER_ID`:

```shell
direnv exec . just check-discord-auth
direnv exec . just plan-example
direnv exec . just apply-example
```

The provider receives a bot token without the `Bot ` prefix. The repository
uses `TF_VAR_DISCORD_BOT_TOKEN` as the semantic environment variable at the
OpenTofu boundary.

The live example currently needs the bot to have these target-server
permissions:

- `Manage Channels`
- `Manage Roles`
- `View Channels`
- `Read Message History`
- `Send Messages`
- `Embed Links`
- `Create Public Threads`
- `Send Messages in Threads`

`Administrator` also satisfies the preflight, but it is broader than the basic
example needs.

The basic role is intentionally an identity label with no guild-level
permissions. Publisher capability is granted by the `papers` channel permission
override. Role paths still require the bot role to be high enough in the
Discord role hierarchy.

Runtime discussion posts use the same bot token and target server variables:

```shell
direnv exec . just create-discussion-post
```

The command creates or reuses an active discussion thread/post by exact title
and records a receipt under `.run/receipts`.

## Documentation Audit

The intended audit template command is:

```shell
direnv exec . just docs-audit-template
```

The local documentation topology is declared in `docs/topology.toml`. The
path-first contract is:

- category directories must match declared `AC` categories;
- durable note files use `AC.ID_semantic_name.md`;
- new categories require a reviewed `docs/topology.toml` diff;
- generated documentation changes must be reviewable before implementation
  plans depend on them.

## Current Tooling Gap

At the time this skeleton was created, the current devenv did not expose a
`wendao` binary on `PATH`. The command above remains the governance target; the
repo should later add a local recipe or package dependency that makes it
available.
