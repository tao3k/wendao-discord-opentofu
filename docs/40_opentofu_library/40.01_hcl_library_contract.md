# OpenTofu HCL Library Contract

The HCL library is the durable infrastructure layer for Knowledge Disco. It
keeps Discord structure auditable, reproducible, and reviewable.

## Current Modules

| Module | Path | Responsibility |
|---|---|---|
| `knowledge_space` | `opentofu/discord/modules/knowledge_space` | Category, text channels, forum channels, roles, and permission overrides. |
| `paper_announcement` | `opentofu/discord/modules/paper_announcement` | Durable Discord paper message with structured embed fields. |

## Provider Contract

The Discord provider is pinned to OpenTofu Registry source
`lucky3028/discord` version `2.6.0`.

Example workspaces should keep `.terraform.lock.hcl` under version control so
provider checksums remain reproducible.

## Module Source Contract

External consumers should pin this repository by Git revision:

```hcl
module "knowledge_space" {
  source = "git::https://github.com/tao3k/wendao-discord-opentofu.git//opentofu/discord/modules/knowledge_space?ref=<commit-sha>"

  server_id = var.server_id
}
```

## State Boundary

HCL should manage stable structure and durable announcements. It should not own
high-frequency chat, exploratory replies, or short-lived discussion state unless
that state is intentionally promoted to a durable governance object.

Forum channels are HCL-managed durable structure. Individual forum posts and
text-channel threads currently live on the runtime side because the pinned
Discord provider has channel and message resources but no thread/post resource.
The runtime command `direnv exec . just create-discussion-post` creates or
reuses a paper discussion post/thread and emits a receipt.
