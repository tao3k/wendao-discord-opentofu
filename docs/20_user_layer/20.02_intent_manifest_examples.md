# Intent Manifest Examples

Intent manifests are reviewable user-layer inputs. They are not provider
schema. Their purpose is to give Agents and humans the same compact language
before changes become HCL variables, runtime Discord actions, or receipts.

## Knowledge Disco

```toml
kind = "knowledge_disco"
name = "Wendao Knowledge"
server_ref = "discord://server/example"

[space]
category = "Wendao Knowledge"
default_channels = ["papers", "paper-discussion", "paper-discussion-forum", "knowledge"]

[policy]
destructive_changes = "human_approval_required"
durable_messages = "hcl_managed"
runtime_discussion = "agent_runtime_with_receipt"
```

## Paper Announcement

```toml
kind = "paper_announcement"
target = "papers"
title = "Attention Is All You Need"
url = "https://arxiv.org/abs/1706.03762"
summary = "A transformer architecture paper."
authors = ["Ashish Vaswani", "Noam Shazeer", "Niki Parmar"]
tags = ["transformer", "attention", "paper"]
discussion = "thread_preferred"
durability = "managed"

source_refs = [
  "wendao://paper/attention-is-all-you-need"
]
```

## Discussion Post

```toml
kind = "discussion_post"
target = "paper-discussion-forum"
paper_ref = "wendao://paper/attention-is-all-you-need"
title = "Discussion: Attention Is All You Need"
prompt = "What is still hard to reproduce or apply from this paper?"
mode = "runtime"
receipt_required = true

source_refs = [
  "wendao://paper/attention-is-all-you-need"
]
```

## Space Management

```toml
kind = "space_management"
scope = "paper-discussion"
request = "The channel is too noisy; propose cleanup and topic rules."

[review]
mode = "proposal_first"
apply_requires_human = true
```

## Field Posture

The first durable contract should stay intentionally small:

- `kind` selects the intent family;
- `target` names a semantic channel key, not a Discord snowflake;
- `source_refs` points back to Wendao or other knowledge context;
- `durability` decides whether the object belongs in HCL;
- `receipt_required` marks runtime work that must be recorded.
