# Intent Manifest Examples

Intent manifests are reviewable user-layer inputs. They are not provider
schema. Their purpose is to give Agents and humans the same compact language
before changes become HCL variables, runtime Discord actions, or receipts.

## Knowledge Operation

```toml
kind = "knowledge_operation"
goal = "publish_paper_with_discussion"
title = "Publish and discuss Attention Is All You Need"
requested_by = "human"
review_mode = "plan_then_apply"
receipt_required = true

[workflow]
engine = "wendao"
model = "bpmn"
process_ref = "wendao://workflow/publish-paper-with-discussion"
run_ref = "wendao://workflow-run/example"
graph_ref = "wendao://workflow-graph/example"
node_ref = "bpmn:SubProcess.PublishPaperWithDiscussion"

[context]
server_ref = "discord://server/wendao"
source_refs = ["wendao://paper/attention-is-all-you-need"]

[[actions]]
id = "announce_paper"
kind = "paper_announcement"
mode = "opentofu"
target = "papers"
intent_path = "examples/intents/paper_announcement.toml"
workflow_node_ref = "bpmn:ServiceTask.AnnouncePaper"
graph_node_ref = "petgraph:announce_paper"

[[actions]]
id = "open_discussion"
kind = "discussion_post"
mode = "runtime"
target = "paper-discussion-forum"
intent_path = "examples/intents/discussion_post.toml"
workflow_node_ref = "bpmn:ServiceTask.OpenPaperDiscussion"
graph_node_ref = "petgraph:open_discussion"
depends_on = ["announce_paper"]

[[actions]]
id = "record_receipt"
kind = "projection_receipt"
mode = "record"
workflow_node_ref = "bpmn:ServiceTask.RecordDiscordReceipt"
graph_node_ref = "petgraph:record_receipt"
depends_on = ["announce_paper", "open_discussion"]
```

## Field Posture

The first durable contract should stay intentionally small:

- `kind` selects the intent or Discord projection envelope family;
- `workflow` carries Wendao BPMN, workflow-run, and petgraph refs;
- `vertical_community_profile` names reusable domain-specific community shapes;
- `target` names a semantic channel key, not a Discord snowflake;
- `source_refs` points back to Wendao or other knowledge context;
- `durability` decides whether the object belongs in HCL;
- `receipt_required` marks runtime work that must be recorded.

Use `knowledge_operation` when Wendao workflow asks this repository to project
one or more Discord actions. Use the smaller intent manifests when the operation
has already been decomposed by Wendao or the Agent.
