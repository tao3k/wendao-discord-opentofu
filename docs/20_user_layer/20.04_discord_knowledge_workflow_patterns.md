# Discord Knowledge Workflow Patterns

Discord is the community interaction surface for Wendao knowledge workflows. It
should not become the workflow authority or the knowledge source of truth. BPMN
and petgraph live in Wendao; this repository projects selected workflow nodes
into Discord structures, posts, roles, events, and receipts.

## Capability Map

| Discord capability | Knowledge workflow role | Projection owner |
|---|---|---|
| Server | Community boundary for one organization or field. | HCL |
| Category | Domain, program, cohort, or project space. | HCL |
| Text channel | Stream discussion, inbox, notification, loose chat. | HCL plus runtime |
| Forum channel | Structured work objects such as papers, agenda items, or cards. | HCL |
| Forum post | One focused knowledge object or workflow item. | Runtime |
| Thread | Subdiscussion, question branch, or refinement loop. | Runtime |
| Role | Participation level, reviewer status, or workflow authority. | HCL |
| Permission | Policy gate and controlled action boundary. | HCL |
| Scheduled event | Time-bound meeting, reading session, or review. | Runtime |
| Reaction or poll | Lightweight signal for priority, vote, or consensus. | Runtime |
| Pin or announcement | Promoted durable context. | HCL or runtime receipt |
| Webhook or bot | Wendao workflow projection executor. | Runtime |

## Pattern: Paper Workflow

Purpose:
Publish a paper, open a focused discussion, collect community feedback, and
return receipts to Wendao.

Discord projection:

- `papers` text channel receives the durable announcement;
- `paper-discussion-forum` receives one forum post per paper;
- optional threads hold narrow questions or replication attempts;
- `knowledge` receives the curated summary after discussion.

Wendao ownership:

- BPMN owns enrichment, review, posting, summarization, and archive gates;
- petgraph owns action dependencies and retry/refinement topology;
- receipts close each Discord projection action.

## Pattern: Agenda Workflow

Purpose:
Turn workflow time into visible community coordination: reading sessions,
research meetings, review windows, and office hours.

Discord projection:

- `agenda` forum channel stores agenda items as forum posts;
- Discord scheduled events represent time-bound sessions;
- agenda post replies collect questions before and after the session;
- completed sessions produce minutes or digest receipts.

Wendao ownership:

- BPMN owns proposal, scheduling, session state, and follow-up gates;
- petgraph connects source materials, agenda items, scheduled events, minutes,
  and follow-up tasks.

Suggested forum tags:

- `proposed`
- `scheduled`
- `in-session`
- `follow-up`
- `done`

Executable projection:

- intent example: `examples/intents/agenda_item.attention_reading_session.toml`;
- scheduled event example: `examples/intents/scheduled_event.attention_reading_session.toml`;
- operation example: `examples/intents/knowledge_operation.agenda_session.toml`;
- runtime command: `direnv exec . just create-agenda-item`;
- scheduled event command: `direnv exec . just create-scheduled-event`;
- receipt example: `examples/intents/receipt.agenda_item.toml`.

## Pattern: Kanban Workflow

Purpose:
Expose workflow state to the community without making Discord the canonical task
database.

Discord projection:

- `kanban` forum channel acts as the visible board;
- one forum post represents one card;
- forum tags mirror state such as `todo`, `doing`, `review`, `blocked`, `done`;
- replies hold implementation discussion or review evidence.

Wendao ownership:

- BPMN owns state transitions and approvals;
- petgraph owns dependencies between tasks, papers, agenda items, and receipts;
- Discord tags are display state, not authority.

Executable projection:

- intent example: `examples/intents/kanban_card.replication_task.toml`;
- operation example: `examples/intents/knowledge_operation.kanban_card.toml`;
- runtime command: `direnv exec . just create-kanban-card`;
- receipt example: `examples/intents/receipt.kanban_card.toml`.

Current provider note:

The pinned `lucky3028/discord` provider version exposes forum channels but does
not expose forum tag management in `discord_forum_channel`. Tags therefore stay
runtime display state for now: profiles declare desired tags, `sync-forum-tags`
creates or preserves them through the Discord API, and work-item receipts record
requested/applied/missing tags.

The same provider version does not expose scheduled events. Scheduled events are
therefore runtime projection objects created through the Discord API and linked
back to agenda receipts.

## Pattern: Knowledge Digest Workflow

Purpose:
Promote useful Discord discussion back into Wendao knowledge.

Discord projection:

- Agent reads a forum post or thread;
- Agent renders a compact digest preview;
- human reviewer approves or edits;
- `knowledge` receives the published digest;
- receipt links digest, source Discord objects, and Wendao refs.

Wendao ownership:

- BPMN owns digest approval and publication state;
- Wendao stores the canonical digest, references, and evidence links.

## Pattern: Vertical Community Bootstrap

Purpose:
Let a Wendao user create a knowledge community for a field such as healthcare,
business, engineering, education, or policy without hand-writing provider
schema.

Discord projection:

- a domain profile selects default channels, forum channels, roles, and review
  posture;
- HCL creates durable structure;
- runtime commands create initial agenda, kanban, or welcome posts;
- receipts record what was created and which profile drove it.

Profile examples:

- healthcare: stricter review, no personalized advice, evidence-first forums;
- business: market intelligence, decision agenda, deal or project kanban;
- research lab: papers, experiments, replication, reading sessions;
- education: cohorts, assignments, office hours, resource digest.

## Design Rules

- Discord is the participation layer.
- Wendao workflow is the process authority.
- Wendao knowledge storage is the durable knowledge authority.
- HCL owns stable Discord structure.
- Runtime owns dynamic Discord interaction objects.
- Receipts join Discord facts back to Wendao BPMN and petgraph state.
- Compact Markdown previews are the primary Agent-facing review surface.
