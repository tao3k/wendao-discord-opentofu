# Discord Interaction Commands

Discord interaction commands are the human control surface for Wendao
workflows. They should produce reviewable intents and receipts, not mutate
Wendao knowledge state directly.

## Command Families

| Command | Purpose | Output |
|---|---|---|
| `/wendao paper` | Submit a paper for announcement and discussion. | `knowledge_operation.publish_paper_with_discussion` |
| `/wendao agenda` | Create an agenda item and optional scheduled event. | `agenda_item` plus `scheduled_event` |
| `/wendao kanban` | Create or update a visible workflow card. | `kanban_card` |
| `/wendao digest` | Promote discussion into a reviewed knowledge digest. | digest review intent |
| `/wendao receipt` | Show projection receipts for a Discord object. | compact receipt summary |

## Interaction Components

Buttons should represent workflow decisions:

- `Approve`
- `Needs Review`
- `Create Follow-up`
- `Promote to Knowledge`
- `Record Receipt`

Select menus should represent bounded choices:

- workflow family;
- target channel or forum;
- reviewer role;
- kanban state;
- agenda session type.

Modals should collect structured fields only when free-form chat is too loose:

- paper URL, title, and summary;
- agenda objective and time;
- kanban acceptance criteria;
- digest title and evidence refs.

## Boundary

Interaction commands are runtime projections. They may create Discord messages,
threads, scheduled events, and receipts. They must not become the canonical
workflow engine; Wendao BPMN and petgraph remain the authority.

The first implemented advanced runtime is scheduled events:

```shell
direnv exec . just create-scheduled-event
```

The command reads `examples/intents/scheduled_event.attention_reading_session.toml`,
creates or reuses a Discord scheduled event, and writes a projection receipt.
