# Healthcare Governance Model

The healthcare profile needs a management surface in addition to the public
knowledge surface. Healthcare managers are not only arranging channels; they
are protecting review quality, privacy posture, evidence provenance, and
workflow accountability.

## Roles

| Role | Responsibility |
|---|---|
| `Clinical Reviewer` | Approves medical claims, evidence summaries, and digest promotion. |
| `Privacy Steward` | Checks that case discussion and digests avoid PHI and personal advice. |
| `Healthcare Moderator` | Handles unsafe advice, off-topic content, and escalation. |
| `Program Manager` | Tracks agenda, review backlog, kanban, and weekly operating rhythm. |
| `Agent Operator` | Reviews failed projections, receipts, and runtime automation health. |
| `Healthcare Maintainer` | Owns durable structure, roles, and high-risk permission changes. |

## Governance Surface

The HCL profile projects a private governance category:

```text
Healthcare Governance
  review-queue
  policy-decisions
  incident-review
  moderation-log
  agent-ops
  privacy-checklist
```

`review-queue`, `policy-decisions`, and `incident-review` are forum channels
because each item is a durable work object. `moderation-log`, `agent-ops`, and
`privacy-checklist` are text channels because they work as streams or operating
logs.

The generated HCL denies `@everyone` from viewing governance channels and grants
access to reviewer, maintainer, privacy, moderator, program, and agent-operator
roles. Wendao remains the canonical workflow authority; Discord is the
management cockpit.

## Review Gate

A `review_gate` is the first management workflow object. It represents a human
decision point before a Discord discussion becomes durable Wendao knowledge.

Example:

```toml
kind = "review_gate"
target = "review-queue"
title = "Review digest: Attention Is All You Need discussion"
review_status = "pending-review"
decision_options = ["approve", "needs_revision", "reject", "promote_to_knowledge"]
```

The runtime command is:

```shell
direnv exec . just sync-forum-tags
direnv exec . just create-review-gate
```

The command creates or reuses a forum post in `review-queue`, copies BPMN and
petgraph refs into the receipt, and records requested/applied/missing display
tags. `sync-forum-tags` prepares tags such as `pending-review`,
`needs-revision`, `approved`, and `promote-to-knowledge` on the governance
forums. Buttons and modals can later bind to the same intent shape.

## Healthcare Safety Defaults

This repository does not implement HIPAA compliance. It sets conservative
workflow defaults for a Discord-based community surface:

- do not post PHI or patient-identifiable data in Discord;
- case review is deidentified-only;
- medical claims require evidence references;
- digest publication requires human review;
- moderation and privacy incidents write receipts.

These defaults align with the general risk posture of minimizing sensitive data
exposure and separating Discord interaction from Wendao canonical storage.
